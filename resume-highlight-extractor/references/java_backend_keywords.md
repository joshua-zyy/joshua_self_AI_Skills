## Java Ecosystem & Frameworks
- **Core Java**: JVM Internals, JMM (Java Memory Model), GC (G1, ZGC, CMS), ClassLoading, Bytecode Manipulation (ASM, Javassist), Reflection, Generics, Lambdas & Streams, SPI (Service Provider Interface).
- **Spring Ecosystem**: Spring Boot, Spring Cloud (Gateway, OpenFeign, Config, Bus), Spring Security (OAuth2, JWT), Spring AOP, Spring Data JPA/Redis, Spring Batch, WebFlux (Reactor).
- **ORM & Database**: MyBatis/MyBatis-Plus, Hibernate/JPA, HikariCP, ShardingSphere, Liquibase/Flyway.
- **Concurrency**: `java.util.concurrent` (ExecutorService, CompletableFuture, CountDownLatch, CyclicBarrier, Semaphore), AQS, CAS, Atomic Classes, ThreadLocal, Fork/Join, Project Loom (Virtual Threads).
- **Microservices & RPC**: Dubbo, gRPC, Nacos, Eureka, Sentinel, Resilience4j, Seata (Distributed Transactions).
- **Middleware**: RocketMQ, Kafka, RabbitMQ, Redis (Redisson, Jedis, Lettuce), ZooKeeper, Elasticsearch.
- **Tools & DevOps**: Maven, Gradle, Arthas, JProfiler, VisualVM, Jenkins, Docker, Kubernetes, Prometheus, Grafana, SkyWalking.

## Architectural Patterns
- **Design Patterns**: Singleton, Factory, Builder, Proxy (Dynamic Proxy, CGLIB), Strategy, Template Method, Observer, Adapter, Decorator, Chain of Responsibility.
- **System Architecture**: DDD (Domain-Driven Design), Hexagonal Architecture, CQRS, Event Sourcing, TCC (Try-Confirm-Cancel), Microservices, Service Mesh.

## Performance & Reliability Concepts
- **JVM Tuning**: Heap sizing (-Xms, -Xmx), GC logging, Dump Analysis (OOM, Deadlock), JIT Optimization, Escape Analysis.
- **Database**: Index Optimization (B+ Tree), Slow Query Analysis, Connection Pooling, Master-Slave Replication, Read-Write Separation.
- **High Concurrency**: Rate Limiting (Token Bucket, Leaky Bucket), Distributed Locking (Redisson), Caching Strategies (Cache Aside, Read/Write Through, Local Cache - Caffeine/Guava), Idempotency Design.
- **Stability**: Circuit Breaking, Fallback, Graceful Shutdown, Warm-up, Stress Testing.

## Metrics to Quantify
- **Latency**: Reduced API latency from Xms to Yms (p99).
- **Throughput**: Supported peak traffic of X QPS/TPS.
- **Resource Usage**: Reduced JVM Heap usage by X%; Reduced CPU load by Y%.
- **Stability**: Achieved 99.99% availability during high-traffic events (e.g., Double 11).
- **Efficiency**: Reduced build time by X%; Optimized startup time from Xs to Ys.
